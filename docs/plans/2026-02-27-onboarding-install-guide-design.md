# 온보딩 앱 설치 안내 화면 디자인

## 개요

온보딩 마지막 단계에 슬라이드형 카드 2장을 추가하여, 앱 설치 방법 안내 + 주요 기능 소개를 제공한다.

## 온보딩 전체 플로우

```
팀 선택(find/create) → 프로필 설정 → [카드 1: 앱 설치] → [카드 2: 기능 소개 + 완료]
```

- 기존 `mode` 상태에 `install` 단계 추가
- `profile` 완료 시 `install` 모드로 전환
- 카드 2에서 "시작하기" 누르면 기존 `completeOnboarding()` 실행

## 카드 1: 앱 설치 안내

### Android (beforeinstallprompt 존재 시)

**제목**: "홈 화면에 추가하면 팀 소식을 바로 확인할 수 있어요"

**혜택 목록**:
- 경기 일정 알림
- 팀원 훈련 일지 알림
- 앱처럼 빠르게 실행

**CTA**: "앱으로 설치하기" (primary) → `usePWA().installApp()` 호출
**Secondary**: "나중에 할게요" (text link) → 카드 2로 이동

### iOS (네이티브 프롬프트 불가)

**제목**: 동일

**단계별 안내**:
1. 하단 공유 버튼(⎋) 탭
2. "홈 화면에 추가" 선택
3. "추가" 탭

- 각 단계에 Safari UI 스크린샷 또는 간단한 일러스트 포함
- CTA: "다음" → 카드 2로 이동
- Secondary: "건너뛰기" → 카드 2로 이동

### 이미 설치된 경우

`usePWA().isInstalled()` === true → 카드 1 자동 스킵, 카드 2부터 시작

## 카드 2: 기능 소개 + 완료

**제목**: "{팀이름} 라커룸에 오신 걸 환영합니다!"

**기능 목록** (아이콘 + 1줄 설명):
- 훈련 일지를 남기고
- 팀원들의 기록을 구경하고
- MVP 투표에 참여하세요

**CTA**: "시작하기 →" (primary) → `completeOnboarding()` 실행 (푸시 구독 포함)

## UI/인터랙션

- 페이지 하단에 도트 인디케이터 (● ○ / ○ ●)
- 좌우 스와이프로 카드 전환 가능
- 카드 전환 시 부드러운 슬라이드 애니메이션
- 각 카드 상단에 일러스트/아이콘 영역

## 플랫폼 감지

- `usePWA` 훅의 `canInstall()` 또는 `navigator.userAgent`로 iOS/Android 분기
- `deferredPrompt` 존재 여부로 Android 네이티브 설치 가능 여부 판단

## 건너뛴 사용자 재유도 (후속 과제)

- 피드 상단 배너: "앱으로 설치하면 알림을 놓치지 않아요" (dismissable)
- 마이페이지/설정에 "앱 설치하기" 항목 추가
- 이 기능은 별도 이슈로 관리
