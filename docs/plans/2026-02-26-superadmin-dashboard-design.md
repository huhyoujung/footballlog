# 슈퍼어드민 대시보드 설계

## 개요

앱 개발자(서비스 오너)가 전체 유저·팀·활동 현황을 한눈에 파악할 수 있는 읽기 전용 대시보드.

## 접근 제어

- URL: `/admin/[secret]`
- 환경변수 `ADMIN_SECRET`과 URL의 `[secret]`이 불일치하면 → `notFound()` (404)
- 로그인 이메일이 `SUPER_ADMIN_EMAIL` 환경변수와 다르면 → `notFound()` (404)
- 존재 자체를 숨기는 방식 (unauthorized 대신 404)

## 환경변수

```
ADMIN_SECRET=랜덤문자열      # URL 시크릿 토큰
SUPER_ADMIN_EMAIL=이메일     # 접근 허용 이메일
```

## 기술 스택

- 서버 컴포넌트 (SSR, 캐싱 없음)
- Prisma 직접 조회
- 별도 API 엔드포인트 없음
- 새로고침으로 최신 데이터 확인

## 페이지 구조

`src/app/admin/[secret]/page.tsx`

## 지표 섹션

### 1. 상단 요약 카드
- 총 유저 수
- 오늘 가입자 수
- 미가입자 수 (team이 null인 유저)
- 총 팀 수
- DAU (오늘 session이 있는 유저 수)

### 2. 최근 가입자
- 컬럼: 이름, 이메일, 가입일, 소속팀
- 최근 20명

### 3. 팀별 인원 현황
- 컬럼: 팀명, 인원수, 최근 훈련 로그 작성일
- 전체 팀 목록

### 4. 최근 7일 훈련 로그
- 날짜별 로그 수 (테이블)

### 5. 활동 없는 유저
- 가입 후 7일 이상 경과했지만 훈련 로그가 0개인 유저
- 컬럼: 이름, 이메일, 가입일, 소속팀

### 6. 최근 피드백
- 앱 내 피드백 목록
- 컬럼: 내용, 작성자, 작성일
